# Refined Lab UI Design

## Goal

美化现有文件发布下载系统，同时保持当前轻量架构：公共端和后台端继续使用单文件 `Vue CDN + Tailwind CDN`，后端 API 不改动，不引入构建工具。

## Visual Direction

采用“精密实验室”风格：暖白底色、墨绿色主视觉、铜色/琥珀色强调，整体克制、可信、适合硬件研发资源分发场景。界面应像一个专业资源库，而不是营销页、网盘或泛蓝色后台模板。

设计系统参考 `ui-ux-pro-max` 的 Trust & Authority、Data-Dense Dashboard、Accessibility 建议，但避免通用蓝灰模板和默认 Inter 风格。字体倾向使用 `Fira Sans` 负责 UI 可读性，`Fira Code` 负责版本号、MD5、Git ID、大小等技术信息。

## Scope

改造两个页面：

- `frontend/public/index.html`：公共下载端。
- `frontend/admin/index.html`：后台管理端。

保持功能不变：

- 公共端目录浏览、搜索、历史版本、下载进度、暗色模式。
- 后台登录、资源管理、上传、成员授权、微信通知、前端配置、安全中心。

## Public Page Design

公共端应突出“资源索引”和“下载可信度”：

- 顶部改为更稳的资源站头部，包含站点标题、副标题、后台入口、暗色切换。
- 面包屑和搜索区更清楚，搜索框保留全局检索感。
- 文件夹和文件行改成更精细的表格/列表样式，减少大圆角和过重阴影。
- 文件信息使用稳定层级：文件名最大，版本/Git/日期/发布人次之，MD5 和 changelog 更弱。
- 历史版本弹窗延续同一视觉语言。

## Admin Page Design

后台应像“发布控制台”：

- 左侧导航更沉稳，使用墨绿主色和清晰选中态。
- 顶部栏保留当前用户、预览公共端、暗色切换，但降低视觉噪音。
- 资源表格增强扫描效率：清晰行高、技术信息等宽字体、操作按钮统一图标按钮样式。
- 上传区保留拖拽体验，视觉上更像“发布仪表面板”。
- 用户、微信、配置、密码页面统一表单卡片、标签、按钮、提示文字风格。

## Interaction And Accessibility

- 所有可点击元素需要有 `cursor-pointer`、明显 hover、focus ring。
- 表格外层应支持小屏横向滚动，避免移动端溢出。
- 图标继续使用 Remix Icon，避免 emoji 图标。
- 动画控制在 150-300ms，避免 hover 导致布局位移。
- 文本对比度优先，浅色模式下正文不使用过浅灰。
- 暗色模式保持可读，但不把页面做成纯黑主题。

## Non-Goals

- 不改后端接口、鉴权逻辑、数据模型。
- 不拆分 Vue 组件，不引入 Vite、React、构建流程或新依赖包。
- 不新增统计卡片、审批流、上传队列等业务功能。
- 不处理现有安全问题，除非样式改造必须触及。

## Verification

实施后需要：

- 静态检查两个 HTML 文件结构没有明显断裂。
- 启动或复用 Docker/本地服务，在浏览器检查公共端和后台端。
- 至少查看桌面宽度和移动宽度，确认没有横向溢出、文字重叠、弹窗不可用。
- 检查浅色和暗色模式。
